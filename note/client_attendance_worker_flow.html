Client Attendance Worker Flow

ภาพรวมการทำงานของระบบกล้องหน้าร้าน ตั้งแต่ Python worker เปิดกล้อง ตรวจจับใบหน้าในเครื่อง ส่ง face crop ไป backend, backend ค้น Rekognition collection เดิม และบันทึกเหตุการณ์เข้า/ออกลง DB โดย ไม่ index ใบหน้าใหม่ และ ไม่เก็บ raw image

Camera Source Notebook camera M4 เป็น entry, M1 เป็น exit ได้
Recognition Search only ใช้ `SearchFacesByImage` เท่านั้น
Transport Multipart POST ส่งเฉพาะ JPEG face crop
Storage Event + Visit เก็บ metadata/hash ไม่เก็บภาพ

System Actors

แต่ละส่วนมีหน้าที่แยกกันชัดเจน: worker รับผิดชอบกล้องและ frame selection, backend รับผิดชอบ credential, Rekognition และ DB state

01

Camera

กล้องจาก MacBook หรือ device จริงในอนาคต เป็นแหล่งภาพเท่านั้น

02

Python Worker

detect หน้า, crop, throttle และส่ง request เข้า backend

03

Next Backend

validate API key, ตรวจขนาดรูป, เรียก service layer

04

AWS Rekognition

ค้น face collection เดิมด้วย `SearchFacesByImage`

05

PostgreSQL

บันทึก attendance event และ visit session

End-to-End Flow

Flow นี้เริ่มจาก worker process หนึ่งตัวต่อกล้องหนึ่งตัว และจบที่การสร้าง/ปิด visit ของลูกค้าที่รู้จักในระบบแล้ว

Step 01
Load worker config
Worker อ่านค่า `.env`: backend URL, API key, camera ID, direction, camera index, throttle interval และ JPEG quality
ATTENDANCE_BACKEND_URL
ATTENDANCE_CAMERA_ID
ATTENDANCE_DIRECTION
Step 02
Open camera stream
Python เปิดกล้องด้วย `cv2.VideoCapture(camera_index)` แล้วอ่าน frame ต่อเนื่องจาก notebook camera หรือกล้อง USB
cameraIndex = 0
preview window
Step 03
Detect face locally
Worker ใช้ OpenCV Haar cascade หาใบหน้าใน frame และเลือกหน้าใหญ่สุด เพื่อลด noise จากฉากหลังหรือคนที่อยู่ไกล
largest face box
minFaceSize
Step 04
Throttle and crop
ถ้ามีหน้าและครบ interval แล้ว worker จะ crop พร้อม padding, encode เป็น JPEG แล้วส่งเฉพาะ face crop ไม่ส่งทุก frame
request every 3s
JPEG crop only
Step 05
POST to backend
Worker ส่ง multipart request ไปที่ `/api/client-attendance/recognize` พร้อม `x-client-attendance-key` และ metadata ของกล้อง
image
cameraId
direction
capturedAt
Step 06
Validate request
Backend เช็ค API key แบบ timing-safe, validate direction, image type และจำกัดขนาดด้วย `CLIENT_ATTENDANCE_MAX_IMAGE_BYTES`
unauthorized = 401
bad request = 400
Step 07
Search face collection
Service เรียก AWS Rekognition `SearchFacesByImage` ด้วย image bytes และใช้ collection + threshold เดียวกับระบบ enrollment
FaceId
ImageId
Similarity
Step 08
Map FaceId to user
Backend lookup `user_face_profiles.face_id` เพื่อหา user แล้วกรองเฉพาะ account ที่ยัง active และไม่ disabled
recognized
unknown
ignored
Step 09
Insert attendance event
บันทึกทุกผลลัพธ์ลง `client_attendance_events`: camera, direction, decision, matched user, similarity, image hash และ metadata
no raw image
SHA-256 only
Step 10
Apply visit state
ถ้าเป็น `entry` และยังไม่มี open visit จะสร้าง `inside`. ถ้าเป็น `exit` และมี open visit จะปิดเป็น `exited`
client_visits.inside
client_visits.exited
Step 11
Return worker feedback
Backend ส่งผลกลับให้ worker แสดงบน preview window เช่น recognized, user name, similarity และ visit status
event.id
user
visit.status

Runtime Boundaries

ขอบเขตนี้ช่วยให้ PoC คุม cost และ security ได้ โดยแยกหน้าที่ worker, backend และ AWS ออกจากกัน

Worker responsibility

  • เปิดกล้องและอ่าน frame
  • detect/crop ใบหน้าในเครื่อง
  • throttle request ไม่ส่งทุก frame
  • ส่ง API key และ metadata ของ camera

Backend responsibility

  • ถือ AWS credential ฝั่ง server เท่านั้น
  • validate image และ request
  • ค้น Rekognition collection
  • ตัดสินใจ entry/exit state และบันทึก DB

Explicit non-goals

  • ยังไม่ทำ liveness ที่หน้าร้าน
  • ยังไม่ทำ tracking ข้ามกล้อง
  • ยังไม่เก็บภาพ raw หรือ video
  • ยังไม่ index ใบหน้าใหม่จาก worker

DB Write Model

ระบบแยก event log ออกจาก visit session เพื่อให้ audit ได้ และยัง query สถานะลูกค้าที่อยู่ในร้านได้ง่าย

client_attendance_events

immutable event ต่อ recognition attempt หนึ่งครั้ง ใช้ตรวจสอบว่า กล้องเห็นอะไร ตอนไหน และ backend ตัดสินใจอย่างไร

camera_id
กล้องที่ส่ง event
direction
entry / exit / sighting
decision
recognized / unknown / ignored
matched_user_id
user ที่ match ได้
matched_face_id
FaceId จาก Rekognition
image_sha256
hash ของรูป crop

client_visits

inferred session ของลูกค้าในร้าน หนึ่ง user มี open visit ได้แค่ หนึ่งรายการด้วย unique index

user_id
ลูกค้าที่รู้จักแล้ว
status
inside / exited / unknown_exit
entered_at
เวลาเข้าจาก entry event
exited_at
เวลาออกจาก exit event
entry_event_id
event ที่เปิด visit
exit_event_id
event ที่ปิด visit

Env Contract

Next.js backend ต้องมี API key สำหรับ worker และ image size limit ส่วน Python worker แต่ละเครื่อง ต้องตั้ง backend URL, camera ID และ direction ให้ตรงกับตำแหน่งกล้อง 

# Next.js .env
CLIENT_ATTENDANCE_API_KEY=<long-random-secret>
CLIENT_ATTENDANCE_MAX_IMAGE_BYTES=800000

# Worker .env
ATTENDANCE_BACKEND_URL=http://<main-mac-lan-ip>:3000
ATTENDANCE_API_KEY=<same-secret>
ATTENDANCE_CAMERA_ID=entry-camera-01
ATTENDANCE_DIRECTION=entry
ATTENDANCE_CAMERA_INDEX=0
